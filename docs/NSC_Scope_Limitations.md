# ขอบเขตและข้อจำกัดของโปรแกรมที่จะพัฒนา

---

## ขอบเขต

1) **ด้านการตรวจจับอุปกรณ์ป้องกันภัยส่วนบุคคล (PPE Detection):** ระบบ ZENTRA รองรับการตรวจจับอุปกรณ์ป้องกันภัยส่วนบุคคล (Personal Protective Equipment) 5 ประเภทหลัก ได้แก่ หมวกนิรภัย (Helmet), เสื้อกั๊กสะท้อนแสง (Safety Vest), แว่นตานิรภัย (Safety Goggles), ถุงมือ (Gloves) และรองเท้าบูทนิรภัย (Safety Boots) โดยครอบคลุมทั้งสถานะ "สวม" และ "ไม่สวม" รวม 18 ประเภทย่อย พร้อมแจ้งเตือนหัวหน้างานทันทีเมื่อพบการละเมิด

2) **ด้านการตรวจสอบพื้นที่อันตราย (Safety Zone Monitoring):** ระบบรองรับการกำหนดพื้นที่ห้ามเข้าในรูปแบบหลายเหลี่ยม (Polygon) ได้สูงสุด 10 โซน โดยผู้ดูแลระบบสามารถวาดขอบเขตผ่านหน้าจอได้โดยตรง และระบบจะบันทึกพิกัดโซนทั้งหมดถาวรในไฟล์ zones.json เพื่อใช้งานในครั้งถัดไปโดยไม่ต้องกำหนดใหม่

3) **ด้านการตรวจจับความเสี่ยงต่อสุขภาพ (Health Risk Detection):** ระบบวิเคราะห์ท่าทางของพนักงานด้วย 3 วิธีพร้อมกัน ได้แก่ การตรวจจับท่าทางผิดปกติจากสัดส่วนกรอบร่างกาย (Bounding Box Ratio), การตรวจจับการล้มกะทันหันจากความเร็วการเคลื่อนที่ของจุดสำคัญบนร่างกาย (Keypoint Velocity) และการตรวจจับการเดินโซเซล่วงหน้าจากวิถีจุดศูนย์ถ่วง (Center of Gravity Trajectory) เพื่อแจ้งเตือนก่อนเกิดเหตุฉุกเฉิน

4) **ด้านแหล่งสัญญาณภาพและแพลตฟอร์ม:** ระบบรองรับแหล่งสัญญาณภาพ 3 รูปแบบ ได้แก่ (1) กล้องเว็บแคม (Webcam) ที่เชื่อมต่อผ่าน USB สำหรับการติดตั้งแบบเคลื่อนที่หรือทดสอบ (2) กล้องวงจรปิด IP Camera ที่ส่งสัญญาณผ่านโปรโตคอล RTSP (Real-Time Streaming Protocol) สำหรับการติดตั้งถาวรในโรงงาน และ (3) ไฟล์วิดีโอสำหรับทดสอบและพัฒนาโมเดล โดยระบบทำงานบน Windows 11 ทั้งแบบ CPU และ GPU และสามารถนำไป Deploy บน Linux Server ผ่าน Docker Container ได้

5) **ด้านระบบแจ้งเตือนและการหน่วงเวลา (Alert System & Cooldown):** ระบบส่งการแจ้งเตือนผ่าน LINE Official Account (LINE OA) พร้อมภาพถ่ายหลักฐาน โดยแบ่งเป็น 3 ระดับตามความรุนแรง ได้แก่ ระดับ WARNING สำหรับการละเมิด PPE ส่งตรงถึงหัวหน้างาน, ระดับ ALERT สำหรับการบุกรุกพื้นที่อันตราย ส่งถึงฝ่ายความปลอดภัยและหัวหน้างาน และระดับ EMERGENCY สำหรับกรณีพนักงานล้มหรือหมดสติ ส่งถึงทีมฉุกเฉิน ฝ่ายความปลอดภัย และหัวหน้างานพร้อมกัน นอกจากนี้ระบบมีการหน่วงเวลา (Cooldown) ระหว่างการแจ้งเตือนแต่ละครั้ง ได้แก่ WARNING หน่วง 30 วินาที, ALERT หน่วง 20 วินาที และ EMERGENCY หน่วง 15 วินาที เพื่อป้องกันการแจ้งเตือนซ้ำซ้อนในเหตุการณ์เดียวกัน

6) **ด้านประเภทโรงงานเป้าหมาย:** ระบบออกแบบสำหรับโรงงานอุตสาหกรรมที่กำหนดให้พนักงานสวม PPE เป็นข้อบังคับ โดยเหมาะสำหรับ 4 ประเภทหลัก ได้แก่ (1) โรงงานผลิตและประกอบชิ้นส่วน (Manufacturing Plant) เช่น โรงงานยานยนต์ อิเล็กทรอนิกส์ และโลหะ (2) คลังสินค้าและโลจิสติกส์ (Warehouse & Logistics) ที่มีการใช้รถยกและเครื่องจักร (3) งานก่อสร้าง (Construction Site) ที่มีความเสี่ยงจากการตกจากที่สูงและวัสดุตกใส่ และ (4) โรงงานเคมีและปิโตรเคมี (Chemical Plant) ที่ต้องการ PPE ป้องกันสารอันตราย โดยมีเงื่อนไขขั้นต่ำว่าพนักงานต้องสวมหมวกนิรภัยและเสื้อกั๊กสะท้อนแสงเป็นอุปกรณ์บังคับ

## ข้อจำกัด

1) ระบบมีประสิทธิภาพการตรวจจับลดลงในสภาพแสงน้อย แสงสะท้อนจัด หรือในพื้นที่ที่มีฝุ่น ควัน หรือหมอกปกคลุมบริเวณกล้อง เนื่องจากโมเดลได้รับการฝึกสอนบนชุดภาพสภาพแสงปกติ จึงแนะนำให้ติดตั้งกล้องในบริเวณที่มีแสงสว่างเพียงพอและไม่มีสิ่งกีดขวางทัศนวิสัย

2) ในกรณีที่พนักงานถูกบดบังด้วยวัตถุหรือบุคคลอื่นเกินกว่า 50% ของร่างกาย ความแม่นยำของการตรวจจับ PPE และการวิเคราะห์ท่าทางจะลดลง เนื่องจากระบบต้องการมองเห็นจุดสำคัญบนร่างกาย (Keypoints) อย่างน้อย 6 จุด ที่มีค่าความมั่นใจ (Visibility) สูงกว่า 0.40 จึงจะวิเคราะห์ได้ครบถ้วน

3) ในเวอร์ชันปัจจุบันระบบรองรับ PPE 5 ประเภทมาตรฐานสำหรับอุตสาหกรรมหนัก โดยในอนาคตมีแผนขยายการรองรับไปยัง PPE เฉพาะทางเพิ่มเติม เช่น ชุดป้องกันสารเคมีแบบเต็มตัว (Full Body Hazmat Suit), หน้ากากกรองสารพิษ (Respirator), สายรัดนิรภัยสำหรับงานที่สูง (Safety Harness) รวมถึงอุปกรณ์สำหรับโรงงานอาหาร เช่น หมวกตาข่าย (Hairnet) และผ้ากันเปื้อน (Apron) ซึ่งสามารถทำได้โดยการเพิ่มชุดข้อมูล (Dataset) และ Fine-tune โมเดลสำหรับแต่ละอุตสาหกรรมโดยเฉพาะ

4) Docker Desktop บนระบบปฏิบัติการ macOS ไม่รองรับการเข้าถึงกล้อง Webcam USB (/dev/video0) โดยตรง หากต้องการใช้งาน Webcam บน Mac จำเป็นต้องรันระบบผ่าน Python โดยตรงด้วยคำสั่ง `python3.11 main.py` แทนการใช้ `docker compose up app` ในขณะที่การใช้กล้อง IP Camera ผ่าน RTSP สามารถใช้ Docker ได้ตามปกติ

5) ในเวอร์ชันปัจจุบัน ระบบแจ้งเตือนผ่าน LINE OA เท่านั้น ยังไม่มีส่วนแสดงผลแบบ Web Dashboard สำหรับติดตามสถานการณ์แบบ Real-time จากระยะไกล ไม่มีฐานข้อมูล (Database) สำหรับสะสมและวิเคราะห์ประวัติเหตุการณ์ย้อนหลัง และยังไม่รองรับการจัดการผู้ใช้หลายคน (Multi-user Management)

6) ฟังก์ชันการตรวจจับและวิเคราะห์ภาพสามารถทำงานบนเครือข่ายภายใน (Local Network) ได้โดยไม่ต้องเชื่อมต่ออินเทอร์เน็ต แต่ฟังก์ชันการแจ้งเตือนผ่าน LINE OA จำเป็นต้องมีการเชื่อมต่ออินเทอร์เน็ต หากการเชื่อมต่อขัดข้อง ระบบจะพยายามส่งซ้ำ 3 ครั้ง ก่อนระงับการแจ้งเตือนชั่วคราว
