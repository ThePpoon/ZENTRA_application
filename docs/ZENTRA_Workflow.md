# ZENTRA — ขั้นตอนการทำงานของระบบ
**Zone Environment Network Thermal Risk Analysis**
ระบบ AI ตรวจสอบความปลอดภัยในโรงงานแบบ Real-time

---

## ภาพรวมระบบ

ZENTRA รับภาพจากกล้องวงจรปิด วิเคราะห์ผ่าน AI 3 โมดูล แล้วส่งการแจ้งเตือนไปยังผู้รับผิดชอบทาง LINE ทันทีเมื่อตรวจพบความเสี่ยง

---

## ขั้นตอนการทำงาน

### ➊ เริ่มต้นระบบ

1. โหลดการตั้งค่าทั้งหมด (API Keys, เกณฑ์การตรวจจับ, ข้อมูลกล้อง)
2. เปิดระบบแจ้งเตือน LINE OA ในพื้นหลัง
3. เปิดระบบบันทึกสถิติรายวัน
4. เลือกแหล่งภาพ → **Webcam** หรือ **RTSP (กล้องวงจรปิด)** หรือ **ไฟล์วิดีโอ**
5. ❓ เปิดกล้องได้หรือไม่?
   - **ไม่ได้** → แสดงข้อผิดพลาด → หยุดระบบ
   - **ได้** → ดำเนินการต่อ
6. โหลดข้อมูลเขตอันตราย (Safety Zones) ที่บันทึกไว้
7. เริ่มต้น Thread อ่านภาพ (FrameReader) และ Thread วิเคราะห์ภาพ (InferenceWorker)

---

### ➋ วงจรหลัก — ทำซ้ำทุก Frame

**อ่านภาพ**
1. รับ Frame จากกล้อง
2. ❓ ได้รับ Frame หรือไม่?
   - **ไม่ได้** → รอ 0.04 วินาที → วนกลับอ่านใหม่
   - **ได้** → ดำเนินการต่อ

**วิเคราะห์ด้วย AI (ทุก 3 Frame)**
3. ❓ ถึงรอบส่งวิเคราะห์หรือไม่? (ทุก 3 Frame)
   - **ใช่** → ส่ง Frame ไปวิเคราะห์ด้วย YOLOv8 (ตรวจ PPE + ตรวจการล้ม)
   - **ไม่** → ใช้ผลการวิเคราะห์ครั้งก่อน
4. ❓ ผลการวิเคราะห์พร้อมหรือไม่?
   - **พร้อม** → อัพเดตผลการตรวจจับล่าสุด
   - **ยังไม่พร้อม** → ใช้ผลเดิม
5. วาด Bounding Box แสดงผลบนหน้าจอ

---

### ➌ โมดูล 1 — ตรวจสอบ PPE (อุปกรณ์ป้องกันส่วนบุคคล)

**เทคโนโลยี:** YOLOv8m — ตรวจจับ 18 ประเภท (หมวก, เสื้อกั๊ก, แว่นตา, ถุงมือ, รองเท้าบูท และการขาด PPE แต่ละชนิด)

1. รับผลจาก YOLOv8 — ตรวจว่ามีการละเมิด PPE หรือไม่
2. ❓ พบการละเมิด PPE (ไม่สวมหมวก / ไม่ใส่เสื้อกั๊ก ฯลฯ) หรือไม่?
   - **ไม่พบ** → บันทึก Frame ปกติ → ไปโมดูล 2
   - **พบ** → ดำเนินการต่อ
3. บันทึก Frame พร้อม annotation เก็บไว้สำหรับ training
4. ❓ หมดเวลา Cooldown 30 วินาทีแล้วหรือยัง?
   - **ยังไม่หมด** → ข้ามการแจ้งเตือน → ไปโมดูล 2
   - **หมดแล้ว** → ส่ง **LINE ⚠ WARNING** แจ้งหัวหน้างาน พร้อมรูปภาพ → ไปโมดูล 2

---

### ➍ โมดูล 2 — ตรวจสอบเขตอันตราย (Safety Zone)

**เทคโนโลยี:** ByteTrack — ติดตาม ID บุคคลต่อเนื่องข้ามหลาย Frame

1. ❓ มีเขตอันตรายที่กำหนดไว้หรือไม่?
   - **ไม่มี** → ข้ามโมดูลนี้ → ไปโมดูล 3
   - **มี** → ดำเนินการต่อ
2. กรองเฉพาะ Detection ที่เป็น "คน" จากผล YOLOv8
3. ByteTrack อัพเดต ID การติดตามบุคคลทุก Frame
   - คน detection ใหม่ → สร้าง Track ID ใหม่
   - คนเดิม → อัพเดตตำแหน่ง Track เดิม
   - ไม่พบคนเดิม → เก็บ Track ไว้รอ 30 Frame ก่อนลบ
4. ตรวจสอบว่าศูนย์กลางตัวคนอยู่ใน Polygon เขตอันตรายหรือไม่
5. ❓ พบบุคคลอยู่ในเขตอันตรายหรือไม่?
   - **ไม่พบ** → ไปโมดูล 3
   - **พบ** → ดำเนินการต่อ
6. บันทึก Frame พร้อม Track ID เก็บไว้สำหรับ training
7. ❓ หมดเวลา Cooldown 20 วินาทีแล้วหรือยัง?
   - **ยังไม่หมด** → ข้ามการแจ้งเตือน → ไปโมดูล 3
   - **หมดแล้ว** → ส่ง **LINE 🔴 ALERT** แจ้ง Safety Officer + หัวหน้างาน พร้อมรูปภาพ → ไปโมดูล 3

---

### ➎ โมดูล 3 — ตรวจสอบการล้ม / Heat Stroke

**เทคโนโลยี:** MediaPipe Pose — วิเคราะห์ 33 จุดบนร่างกาย (Keypoints)

**ขั้นตอนวิเคราะห์ท่าทาง (ทุก Frame)**
1. แปลงภาพเป็น RGB → ประมวลผลด้วย MediaPipe Pose
2. ❓ ตรวจพบโครงกระดูก (Pose) ได้หรือไม่?
   - **ไม่ได้** → ลดตัวนับการยืนยัน 1 → ไปตรวจสอบการแจ้งเตือน
   - **ได้** → วิเคราะห์ 3 วิธี พร้อมกัน

   **วิธีที่ 1 — BBox Ratio** (ตรวจท่านอน):
   - คำนวณ กว้าง ÷ สูง ของกรอบร่างกาย
   - ❓ กว้าง ÷ สูง > 0.72 (นอนราบ)?
     - ใช่ → ตั้งค่า abnormal_posture = จริง

   **วิธีที่ 2 — Keypoint Velocity** (ตรวจการล้มฉับพลัน):
   - คำนวณค่าเฉลี่ยความเร็วการเคลื่อนที่ของ Keypoints ระหว่าง Frame
   - ❓ ความเร็วเฉลี่ย > 0.30?
     - ใช่ → ตั้งค่า sudden_fall = จริง

   **วิธีที่ 3 — Gait COG** (ตรวจการเดินผิดปกติ):
   - คำนวณความแปรปรวนของจุดศูนย์กลางมวลร่างกาย (ระหว่างสะโพกซ้าย-ขวา) ใน 30 Frame ล่าสุด
   - ❓ ความแปรปรวน > 0.20?
     - ใช่ → แสดงข้อความ "GAIT ANOMALY" บนหน้าจอ (เฝ้าระวัง ไม่ส่ง Alert)

3. ❓ abnormal_posture = จริง **หรือ** sudden_fall = จริง?
   - **ไม่ใช่** → ลดตัวนับการยืนยัน 1
   - **ใช่** → เพิ่มตัวนับการยืนยัน 1 → แสดง "FALL RISK [n/6]" บนหน้าจอ

**ขั้นตอนตัดสินการแจ้งเตือน**
4. ❓ ตัวนับการยืนยัน ≥ 6 Frame ติดต่อกันหรือไม่?
   - **ยังไม่ถึง** → ไปตรวจคีย์บอร์ด
   - **ถึงแล้ว** → รีเซ็ตตัวนับ → ดำเนินการต่อ
5. ❓ หมดเวลา Cooldown 15 วินาทีแล้วหรือยัง?
   - **ยังไม่หมด** → ไปตรวจคีย์บอร์ด
   - **หมดแล้ว** → บันทึก Frame (บังคับ ไม่สนใจ Quota) → ส่ง **LINE 🆘 EMERGENCY** แจ้งทุกกลุ่ม พร้อมรูปภาพ

---

### ➏ ส่งการแจ้งเตือน LINE OA

ระบบแจ้งเตือนทำงานในพื้นหลัง ไม่รบกวนการประมวลผลภาพหลัก

1. รับคำร้องแจ้งเตือน (ระดับ + ข้อความ + รูปภาพ)
2. ตรวจสอบ Cooldown ซ้ำอีกครั้ง (ป้องกัน Alert ซ้ำ)
3. เข้ารหัสรูปภาพ JPEG คุณภาพ 90%
4. อัพโหลดรูปไปยัง catbox.moe → ได้ URL ภาพ
5. ส่งข้อความ + URL ภาพ ไปยัง LINE Group ที่กำหนด
6. ❓ ส่งสำเร็จหรือไม่?
   - **สำเร็จ** → บันทึก Log
   - **ล้มเหลว** → ลองส่งซ้ำสูงสุด 3 ครั้ง (รอ 2 วินาทีต่อครั้ง)

| ระดับ | เหตุการณ์ | Cooldown | กลุ่มที่แจ้ง |
|-------|----------|---------|------------|
| ⚠ WARNING | ไม่สวม PPE | 30 วินาที | หัวหน้างาน |
| 🔴 ALERT | เข้าเขตอันตราย | 20 วินาที | Safety Officer + หัวหน้างาน |
| 🆘 EMERGENCY | ตรวจพบการล้ม | 15 วินาที | ทีม Emergency + Safety Officer + หัวหน้างาน |

---

### ➐ วนซ้ำ / หยุดระบบ

1. ❓ ผู้ใช้กด Q หรือไม่?
   - **ไม่กด** → กลับไปขั้นตอน ➋ วนซ้ำ
   - **กด Q** → ดำเนินการต่อ
2. หยุด Thread อ่านภาพ + Thread วิเคราะห์ภาพ
3. ส่ง Alert ที่ค้างในคิวให้ครบก่อนปิด
4. ปิดกล้อง + ปิดหน้าต่างแสดงผล
5. แสดงสถิติสุดท้าย
6. **จบการทำงาน**

## สรุปสัญลักษณ์สำหรับทำ Flowchart

| สัญลักษณ์ | ใช้กับ |
|----------|-------|
| **วงรี (Oval)** | เริ่มต้น / สิ้นสุด |
| **สี่เหลี่ยม (Rectangle)** | ขั้นตอนการทำงาน |
| **สี่เหลี่ยมขอบคู่** | การเรียก Module / Thread / LINE API |
| **สี่เหลี่ยมขนมเปียกปูน (Parallelogram)** | รับ/ส่งข้อมูล (กล้อง / Frame) |
| **ข้าวหลามตัด (Diamond)** | เงื่อนไข ❓ ใช่/ไม่ใช่ |
| **วงกลมเล็กมีตัวเลข** | Connector วนซ้ำ (เช่น วนกลับขั้นตอน ➋) |

## ลำดับโมดูลการทำงานที่ถูกต้อง

```
วาดภาพ (Drawing)
  → Fall Module on_frame  [MediaPipe วิเคราะห์ท่าทาง]
  → Zone Module on_frame  [วาดเขตอันตราย]
  → PPE  Module on_frame  [แสดง OSD + imshow บนหน้าจอ]

ตรวจจับ (Detection)
  → PPE  Module on_data   [ตรวจ PPE violations]
  → Zone Module on_data   [ตรวจบุคคลในเขตอันตราย]
  → Fall Module on_data   [ตัดสินใจส่ง Alert การล้ม]
```
